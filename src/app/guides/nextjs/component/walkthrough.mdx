## ‚úÖ What Is a Component?

> A component in React (and by extension, Next.js) is a **JavaScript/TypeScript function that returns JSX**.

There are two main types:

### 1. **Server Components** (Default in Next.js `app/` directory)

```tsx
export default function MyComponent() {
	return <div>Hello World</div>;
}
```

- Renders on the server
- No need for `use client`
- Cannot use hooks like `useState`, `useEffect`

### 2. **Client Components** (Interactive/UI-based)

```tsx
"use client";

export default function MyButton() {
	const [clicked, setClicked] = useState(false);
	return <button onClick={() => setClicked(true)}>Click me</button>;
}
```

- Must include `'use client'`
- Allows use of hooks
- Runs in the browser

## üß† Proper Terminology

When you see:

```tsx
export default function AboutPage() {
  return (...)
}
```

This is called a:

### ‚úÖ **Default Exported React Functional Component** (RFC)

- **React**: Uses JSX/TSX and returns UI
- **Functional**: It's a JavaScript function, not a class
- **Component**: Reusable UI building block
- **Default Export**: Exported as the main module

> You may also hear this referred to as: `RFC`, `FC`, `Page Component`, or `App Route Component`
